## 2. 项目现状

这是一个纯前端、单文件的 H5 工具，没有构建系统、没有后端、没有依赖包，核心逻辑都在 `index.html` 里。

项目功能：
- 用户回答固定问题后，生成一张 KOC 视频拍摄任务卡
- 结果包含 summary、opening、script、shots、titles、comment/reply、自检结果等
- 前端直接本地打开 `index.html` 即可运行

## 3. 已完成的关键迭代

最近一轮已经把“主宣传点”和“证据点”重新拆开，避免脚本主题跑偏。

### 主宣传点现在只保留 3 类

- 信息写得更清楚，少踩坑
- 跟 AI 说条件，更快推荐合适岗位
- 选好岗位后，约面试很快

### 证据点现在只作为 supporting evidence

- 工资 / 薪资构成
- 福利 / 包吃住
- 宿舍环境
- 体检要求 / 体检费
- 面试与入职
- 离职条件
- 岗位特点

规则变化：
- 证据点不能再单独抢整条视频主题
- 正文必须先有试用动作，再有页面查看，再有证据点，再给判断
- 不能直接整条围着“体检费/离职条件/宿舍”打转

### 结构与连贯性

- 正文链路已调整为：
  1. `trigger`
  2. `intro`
  3. `action`
  4. `reaction`
  5. `merged evidence/proof`
  6. `judgement`
  7. `recommend`
  8. `close`
- 新增了 `trigger_pool`，用来补“为什么今天会试这个”的触发点
- 之后又把 `trigger_pool` 改成“纯场景铺垫”，尽量避免和 `intro` 连续重复解释“为什么试这个”
- `mergeNarrativeLines(...)` 现在不是一律用逗号硬拼；如果两句太长或后一句本身像新判断，会自动断成两句，减少“焊成长句”的问题
- 正文拼装后新增了一层相邻句开头口头词去重，避免连续几句都以 `反正 / 说真的 / 说实话 / 怎么说呢 / 诶` 起头

### 脚本口语与逻辑

- 针对很多“像写稿子、不像工友说话”的句子做过多轮改写
- 重点收过这些类型：
  - `我没有……而是……`
  - `让我觉得方便的是……`
  - `我比较有感觉的就是……`
  - `这个小程序就叫……`
  - `（闪电狗AI快聘小程序）`
- 最近又专门扫了一轮这类默认前文已经交代过的口头习惯词：
  - `这次我……`
  - `后来才……`
  - `我自己点开……`
  - `我自己跑了一遍……`
  - `一上来……`
- 目前方向是：尽量减少“默认观众已经知道上下文”的说法，让每一句都更像第一次自然讲出来

### 主题内容调整

- `岗位真实_薪资真实`
  - 增加了更具体的“被坑场景”素材，例如工资写得高、去了才发现底薪低、体检费白交、班次和通勤被坑等
  - 去掉了主链路里不符合常识的“去面试先看离职条件”这类表达
- `AI按条件推荐岗位`
  - 强调“先说条件 -> 先筛一轮 -> 再问自己在意的问题 -> 再往下走”
  - 修过“我后面问了两个点，它也不是推完就不管了”这种表达，改成强调“回复快、节省时间”
- `选好岗位后几秒约面_全流程2到3分钟`
  - 改掉了高频的“卡在某一步”说法
  - 当前主要对比的是：以前找中介/打电话/等回复/跑线下很慢；现在 AI 24 小时能回，线上两三分钟就能把约面试走完

### 标题与模块

  - 从碎片短句改成“前半句痛点/场景 + 后半句结果/方法/对比/悬念”
  - 去掉了 `实测 / 体验 / 冒号` 这种过重的测评博主口气
- 页面新增了固定模块 `话题引用`
  - 当前固定内容：

    - `#闪电狗AI`

## 4. 正文体验链路

当前期望的正文逻辑：

1. 我最近试了下闪电狗AI快聘小程序
2. 我先说了几个找工作条件
3. 系统先推荐一批岗位
4. 我继续缩小范围或选中岗位
5. 我再点开职位详情页或相关页面看信息
6. 我重点看工资、福利、体检、宿舍、离职条件这类证据点
7. 最后给出个人判断，必要时补一句互动问题

要求：
- 逐句版默认要像完整口播，不要只剩两三句碎片
- 提纲版也要保持完整链路，不是零散关键词
